Важные сведения об этой версии Microsoft Windows XP

 К началу

Этот выпуск Microsoft® Windows® XP включает пакет обновления 3 (SP3). Это означает, что при установке версии Windows XP Home Edition или Windows XP Professional этого выпуска автоматически устанавливается пакет обновления 3 (SP3).

Если не указано иное, все сведения из этих заметок о выпуске относятся к обеим системам (Windows XP Home Edition и Windows XP Professional).

В документе термин Windows XP используется для обозначения и Windows XP Home Edition, и Windows XP Professional, если речь не идет только об одной из этих операционных систем.

Важные сведения для всех пользователей

  • Установка системы Windows XP описана в разделе "Пошаговые инструкции. Установка Microsoft Windows XP."
  • Дополнительные сведения о пакете обновления см. на веб-узле пакета обновления 3 (SP3) для Windows XP.

Важные сведения для администраторов и опытных пользователей

  • Дополнительные сведения о пакете обновления 3 (SP3), такие как заметки о выпуске и руководство по развертыванию, см. в разделе веб-узла Microsoft TechNet Ресурсы для пакета обновления 3 (SP3) для Windows XP (на английском языке).
  • Сведения об обновлениях, входящих в пакет обновления 2 (SP2) для системы Windows XP, см. в статье 811113 базы знаний Майкрософт, "Список исправлений, включенных в состав пакета обновления 2 (SP2) для Windows XP".
  • Сведения о пакете обновления 2 (SP2), не включенные в заметки о выпуске, см. в статье 835935 базы знаний Майкрософт, "Замечания к выпуску пакета обновления 2 (SP2) для Windows XP".
  • Подробные технические сведения о пакете обновления 2 (SP2), в том числе о совместимости программного обеспечения, см. в статье Пакет обновления 2 (SP2) для Windows XP: ресурсы для ИТ-специалистов (на английском языке) на веб-узле Microsoft TechNet.

Файловые системы

 К началу

Публикация в Интернете и шифрование файлов

Чтобы обеспечить конфиденциальность передаваемых данных, в системе Windows XP используется шифрованная файловая система (EFS) NTFS для локального шифрования файлов. Однако в случае перехода с систем Microsoft Windows 98 или Microsoft Windows Millennium Edition без преобразования файловой системы в NTFS или при выборе в ходе установки файловой системы FAT шифровать файлы на веб-серверах будет невозможно.

Сведения о преобразовании файловых систем в NTFS см. в центре справки и поддержки системы Windows XP.



Сеть и подключения

 К началу

Вопросы совместимости с клиентскими приложениями протокола PPPoE сторонних разработчиков

Этот раздел посвящен проблемам, связанным с использованием приложений виртуальной частной сети сторонних разработчиков для удаленного доступа к компьютерам.

Многие клиентские приложения протокола PPPoE сторонних разработчиков работают или устанавливаются в системе Windows XP неправильно. Если для подключения поставщик услуг Интернета применяет протокол PPPoE, возможные проблемы можно устранить, используя клиент PPPoE в системе Windows XP или запросив клиентское приложение PPPoE, совместимое с Windows XP, у поставщика программного обеспечения или поставщика услуг Интернета.

При использовании одного из следующих продуктов обратитесь к поставщику за версией программного обеспечения, совместимой с системой Windows XP.

  • WinPoET версии 2.1a или более ранней
  • EnterNet 300 версии 1.34 или более ранней

Чтобы создать подключение с помощью клиента PPPoE в составе системы Windows XP, выполните указанные ниже действия.

  1. В меню Пуск выберите пункт Панель управления, а затем — элемент Сеть и подключения к Интернету.
  2. В разделе Сетевые задачи выберите пункт Создание нового подключения.

    Будет запущен мастер новых подключений.

  3. Нажмите кнопку Далее.
  4. Выберите один из перечисленных ниже параметров, подтвердив выбор нажатием кнопки Далее.
    • Подключить к Интернету
    • Установить подключение вручную
    • Через высокоскоростное подключение, запрашивающее имя пользователя и пароль
  5. Следуйте указаниям в мастере.

    Поле Имя поставщика услуг можно оставить пустым. В этом случае система Windows XP попытается обнаружить подключение.

Сетевые протоколы, поддержка которых в системе Windows XP прекращена

Система Windows XP больше не поддерживает некоторые сетевые протоколы, поскольку возможности, которые они обеспечивают, заменены эквивалентными возможностями набора протоколов TCP/IP.

В системе Windows XP прекращена поддержка следующих протоколов:

  • NetBEUI, немаршрутизируемый протокол;
  • DLC (Data Link Control), немаршрутизируемый протокол;
  • AppleTalk, протокол, используемый с компьютерами Apple.

Клиентские приложения Telnet на терминалах VTNT для восточноазиатских языков

Для правильной работы с восточноазиатскими языками настройте клиенты Telnet на терминалах VTNT на использование шрифтов TrueType. Это необходимо сделать до запуска клиента Telnet. Процедура описана ниже.

Чтобы настроить использование шрифтов TrueType в клиентах Telnet, выполните указанные ниже действия.

  1. Откройте окно командной строки.
  2. Щелкните правой кнопкой мыши заголовок или системное меню окна командной строки и выберите команду Свойства.
  3. На вкладке Шрифт выберите шрифт TrueType и нажмите кнопку ОК.
  4. Выберите параметр Изменить ярлык для запуска этого окна и нажмите кнопку ОК.
  5. В меню Пуск выберите команду Выполнить и введите команду:

    telnet
  6. После открытия окна Telnet повторите действия 2—4.

Беспроводные сетевые устройства

Адаптер беспроводной сети ORiNOCO

При установке программ с компакт-диска для адаптера беспроводной сети ORiNOCO служба настройки беспроводной связи Windows XP работает неправильно, так как эти программы вмешиваются в работу элемента управления адаптером. В результате вкладка настройки беспроводной связи в окне свойств сети отсутствует и адаптер не настраивается автоматически.

Чтобы устранить неполадки в работе адаптера беспроводной сети ORiNOCO, выполните указанные ниже действия.

  1. Удалите программы ORiNOCO с помощью элемента панели управления Установка и удаление программ.
  2. Для удаления адаптера ORiNOCO используйте диспетчер устройств.

    Чтобы открыть диспетчер устройств, нажмите кнопку Пуск и выберите команду Панель управления. Выберите элемент Производительность и обслуживание и щелкните значок Система. На вкладке Оборудование нажмите кнопку Диспетчер устройств.

  3. Переустановите драйверы, чтобы повторно определить адаптер с помощью параметра диспетчера устройств Обновить конфигурацию оборудования или путем перезагрузки системы Windows XP.

Сообщения беспроводной сети

Эти сведения относятся к беспроводным устройствам, использующим следующие стандарты:

  • IEEE 802.11 (стандарт беспроводной сети Ethernet);
  • IEEE 802.1X (стандарт проверки управления сетевым доступом на основании порта).

При использовании беспроводной конфигурации IEEE 802.11 и протокола проверки подлинности сети IEEE 802.1X сведения о пользователях и запросы действий отображаются во всплывающих сообщениях в области уведомлений (в правом нижнем углу экрана).

Однако после обновления до Windows XP этот значок может не появляться. Завершив установку, проверьте, появился ли значок. Если он отсутствует, используйте описанную ниже процедуру для устранения этой неполадки.

Чтобы значок сети отображался в области уведомлений, выполните указанные ниже действия.

  1. В меню Пуск выберите пункт Панель управления, а затем — элемент Сеть и подключения к Интернету.
  2. Выберите пункт Сетевые подключения.
  3. Щелкните правой кнопкой мыши значок беспроводного подключения и выберите команду Свойства.
  4. На вкладке Общие установите флажок При подключении вывести значок в области уведомлений.

Антивирусные программы

Убедитесь в том, что используемые антивирусные программы предназначены именно для установленной версии Windows XP. Это значит, что программное обеспечение должно быть создано специально для Windows XP Home Edition или Windows XP Professional.

Драйверы антивирусной программы, не предназначенные для конкретной версии системы Windows XP, могут вызвать неполадки. Например, программное обеспечение может не поддерживать проверку на наличие вирусов в режиме реального времени, что сделает систему более уязвимой для атак.

Неполадки могут быть как незначительными (которые можно устранить), так и серьезными (например, частичная или полная потеря данных или невозможность использовать компьютер).

Чтобы избежать таких неполадок, убедитесь в том, что антивирусная программа предназначена для используемой операционной системы. При возникновении вопросов или необходимости приобрести ту или иную антивирусную программу обратитесь к поставщику программного обеспечения.

Microsoft Agent

Система Windows XP поддерживает программы и модули с интерфейсом SAPI (Speech Application Programming Interface) версии 5.0. Чтобы избежать функциональных сбоев в приложениях Microsoft Agent, использующих речевые модули SAPI 4.0, установите поддержку среды выполнения SAPI 4.0a и переустановите речевые модули SAPI 4.0. Это необходимо сделать, даже если в их работе с Microsoft Agent до обновления до Windows XP не было сбоев.

Чтобы установить поддержку среды выполнения SAPI версии 4.0a, перейдите по следующей ссылке:

  • Установка поддержки среды выполнения SAPI 4.0a

Проверка подлинности SMTP

Эта заметка относится только к версии Windows XP Professional.

В системе Windows XP службу протокола SMTP можно настроить на проверку подлинности клиентов перед использованием ими команд. При установке или обновлении до Windows XP Professional в рабочей группе параметры проверки подлинности NTLM (ранее Windows NT LAN Manager) и Generic Security Services API (GSSAPI) не работают, так как раздел реестра настроен на проверку подлинности только гостевых учетных записей.

Эту проблему можно решить, настроив раздел реестра принудительной проверки подлинности гостевых учетных записей.

Дополнительные сведения см. в статье 304707 базы знаний Майкрософт "Настройка проверки подлинности SMTP в системе Windows XP Professional Edition" (на английском языке).

Внимание!

  • Ошибка при изменении реестра может серьезно повредить систему, поэтому перед его выполнением необходимо создать резервные копии всех важных данных.


Хранение

 К началу

Диспетчер логических томов

Имена локальных дисков (буквы латинского алфавита) обычно назначаются последовательно, начиная с буквы C (C, D, E и т. д.). Для имен сетевых дисков буквы назначаются в обратном порядке, начиная с буквы Z (Z, Y, X и т. д.).

Конфликты между именами дисков и существующих общих сетевых дисков могут возникать при подключении к компьютеру с системой Windows XP цифровых камер и других устройств, использующих карты памяти. То же может происходить при добавлении внешних дисков (например, USB-дисков или дисков 1394). При возникновении конфликта с именем существующего общего сетевого диска устройство или внешний диск становятся недоступны.

Чтобы предотвратить эту проблему, разрешите автоматическое назначение имен сетевых дисков системой Windows или выбирайте для дисков буквы из начала алфавита.

Управление томами

Система Windows XP не поддерживает наборы томов Microsoft Windows NT® 4.0 Ftdisk.

При работе с системой Microsoft Windows 2000 перед обновлением до Windows XP преобразуйте наборы томов Ftdisk в динамические тома. При работе с системой Windows NT 4.0 перед обновлением до Windows XP разделите все зеркальные тома и выполните архивацию данных в наборах чередующихся томов, томов RAID5 или расширенных томов. Наборы Ftdisk могут быть недоступны после обновления.
